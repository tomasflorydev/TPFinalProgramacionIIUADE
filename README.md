# Preguntas teoricas
## 1. ¿Como harias para que el grafo sea dirigido?
La implementación de un grafo dirigido se logra haciendo que la creación de aristas sea unidireccional dentro del método que maneja las conexiones, típicamente llamado `conectarNodo`. Esto requiere que el grafo mantenga un flag booleano que se inicializa al construir el grafo (por ejemplo, `new Grafo(true)`). Luego, dentro del método de conexión, la lógica se bifurca: si el flag es true, solo se agrega la arista del nodo origen al nodo destino (A -> B); si es false (no dirigido), se agrega la arista de retorno automáticamente (B -> A), estableciendo así la diferencia fundamental entre ambos tipos de grafo.
## 2. ¿Se puede recorrer igual?
Sí, un grafo dirigido se puede recorrer con los mismos algoritmos que uno no dirigido, principalmente la Búsqueda en Amplitud (BFS) y la Búsqueda en Profundidad (DFS). La diferencia radica en que los algoritmos de recorrido solo siguen las aristas disponibles en la dirección en la que fueron definidas (A -> B), mientras que ignoran las rutas inversas si no están explícitamente creadas. Esto significa que los caminos que se encuentren serán válidos dentro de la estructura de un dígrafo, y los algoritmos no necesitan ninguna modificación interna para funcionar; simplemente, el resultado reflejará las mayores restricciones en la conectividad impuestas por la dirección .
